Quickstart¶

ここでは Shimehari の使い方について簡単に解説します。

How to use¶

アプリケーションを作成したいディレクトリで以下のコマンドを実行するだけで 最初の画面が表示されます。

$ cd your_proj_directory
$ shimehari create your_app_name
$ shimehari drink
$ * Shimehari GKGK!
$ * Running on http://127.0.0.1:5959/

It works! and Let’s drink!

Debug Mode¶

Shimehari を debug 状態で起動するのはとても簡単です。

$ shimehari drink -d

たったこれだけで Shimehari は debug 状態で起動。

アプリケーションに何かエラーがあったときは Werkzeug のデバッガーがブラウザ上に表示されます。

RESTful Routing & Controller¶

Shimehari は RESTful なルーティングを推奨しています。

Routing¶

Router に対し、Resource としてコントローラークラスを追加するだけで自動的に RESTful なアクションがルーティングされます。

appRoutes = Router([
        Resource(IndexController, root=True),
        Resource(FooController),
        Resource(BarController)
    ])

現在のルーティング状況は以下のコマンドを叩くことで確認することが出来ます。

$ shimehari routes
$ Your Shimehari App Current Routing.

Methods       |URL                          |Action
----------------------------------------------------------------------
GET            /                             [action => index, controller => IndexController]
POST,GET       /<int:id>                     [action => show, controller => IndexController]

rails のように Resource をネストすることで 自動的に URL も1階層深く設定されます。

appRoutes = Router([
        Resource(IndexController, root=True),
        Resource(FooController,[
                Resource(BarController)
                ])
    ])

$ shimehari routes
$ Your Shimehari App Current Routing.

Methods       |URL                          |Action
----------------------------------------------------------------------
GET            /                             [action => index, controller => IndexController]
POST,GET       /<int:id>                     [action => show, controller => IndexController]
GET            /foo                          [action => index, controller => FooController]
GET            /foo/bar                      [action => index, controller => BarController]

Controller¶

shimehari.controllers.ApplicationController を継承することで Resource() を使ってルーティングを拾い上げることが出来ます。 コントローラーで実装されていない RESTful アクションは自動的にルーティングの対象外となります。

from shimehari import ApplicationController
class ExampleController(ApplicationController):
        def index(self, *args, **kwargs):
                return self.renderTemplate('index.html')

ApplicationController を自前で継承後、必要なアクションを定義するだけでなく、 以下のコマンドを叩くことで一通りアクションが定義済みのコントローラーを生成することが出来ます。

$ shimehari generate controller Example

Command¶

Shimehari ではアプリケーションの開発をサポートするため 様々なコマンドラインツールを用意しています。

Create Application¶

Shimehari アプリケーションを作成するために自分自身でファイルを幾つも作成する必要はありません。 create コマンドを実行するだけで Shimehari アプリケーションを動かすために必要なファイルが自動的に生成されます。

$ shimehari create app_name

create コマンドの実行結果として、以下のファイルが生成されます。

/your_proj_dir
  ├ config.py
  └/app_name
    ├ __init__.py
    ├ main.py
    ├ router.py
    ├/log
    ├/assets
    ├/models
    │ └ __init__.py
    ├/views
    └/controllers
      ├ __init__.py
      └ index.py

Development Server¶

開発中は drink コマンドを実行することで開発用サーバーを立ち上げることができます。

$ shimehari drink
$ * Shimehari GKGK!
$ * Running on http://127.0.0.1:5959/

drink コマンドには幾つかのオプションが存在します。 例えば、アプリケーションをデバッグモードで実行したい場合は以下のオプションを付けます。

$ shimehari drink --debug

Generate Module¶

アプリケーションを作る上で必要となる様々なモジュールをコマンドラインから生成することができます。 現状 generate コマンドで生成できるのはコントローラーだけですが、将来的には様々なものが生成できるようになる予定です。

コントローラーの生成は以下のように generate コマンドを実行します。

$ shimehari generate controller controller_name

Routing Map¶

アプリケーションのルーティングがどうなっているのか コマンドラインから確認することができます。

現状のルーティングを確認するには routes コマンドを実行します。

$ shimehari routes
$ Your Shimehari App Current Routing.

Methods       |URL                          |Action
----------------------------------------------------------------------
GET            /                             [action => index, controller => IndexController]
POST,GET       /<int:id>                     [action => show, controller => IndexController]
GET            /foo                          [action => index, controller => FooController]
GET            /foo/bar                      [action => index, controller => BarController]
