Application Object¶

class shimehari.Shimehari(importName, staticURL=None, staticFolder='static', appFolder='app', controllerFolder='controllers', viewFolder='views', assetsFolder='assets', instancePath=None, isRelativeConfig=False, templateOptions={})¶

Shimehari Object は WSGI アプリケーションとして必要な機能を実装しており、 アプリケーションの中心となるオブジェクトです。

メインモジュール、または __init__.py ファイルの中で以下のように書くことで Shimehari インスタンスを生成することができます。

from shimehari import Shimehari
app = Shimehari(__name__)

ただし、Shimehari では通常コマンドラインでアプリケーションを生成することを推奨しているので あなたが直接インスタンス生成のコードを書くことはそうそうないと思われます。

Parameters:
  • importName – アプリケーションのパッケージ名
  • staticURL – サイト内共通、静的ファイルの URL
  • staticFolder – 静的ファイルが格納されているディレクトリ
  • appFolder – アプリケーション全体が格納されているディレクトリ
  • controllerFolder – コントローラーが格納されているディレクトリ
  • viewFolder – ビューが格納されているディレクトリ
  • assetsFolder – アセットファイルが格納されているディレクトリ
  • instancePath – アプリケーションのための代替インスタンスパス

class shimehari.configuration.Config(environment='development', defaults={})¶

Config クラスはアプリケーションの設定を保存します。 初期値として、以下の設定を保持しています。

Parameters:
  • DEBUG –

    アプリケーションをデバッグモードとして起動するかどうか。 この項目が True に指定された場合、アプリケーションは常にデバッグモードで起動されます。 一時的にデバッグモードで起動したい場合は以下のコマンドを用いたほうが便利です。

    $ shimehari drink -d
  • TEST – アプリケーションをテストモードとして起動するかどうか。
  • APP_ROOT – アプリケーションのルートディレクトリを指定します。
  • APP_DIRECTORY – アプリケーションのファイルが置かれたディレクトリを指定します。
  • MAIN_SCRIPT – アプリケーションで一番最初に実行されるスクリプトを指定します。
  • APP_INSTANCE_NAME – アプリケーションがインスタンス化される際の名前を指定します。
  • CONTROLLER_DIRECTORY – コントローラーが格納されたディレクトリを指定します。
  • VIEW_DIRECTORY – ビューファイルが格納されたディレクトリを指定します。
  • MODEL_DIRECTORY – モデルファイルが格納されたディレクトリを指定します。
  • ASSETS_DIRECTORY – アセットファイルが格納されたディレクトリを指定します。
  • PREFERRED_URL_SCHEME – URL を発行する際のプロトコルを指定します。
  • AUTO_SETUP – 自動的にセットアップを開始するかどうか指定します。 この項目についての詳細は app.setup を参照してください。
  • CONTROLLER_AUTO_NAMESPACE – コントローラーディレクトリ内で、サブディレクトリが作成されていた場合、 自動的にサブディレクトリ名を URL に反映させるかどうかを決定します。
  • TEMPLATE_ENGINE – デフォルトで使用するテンプレートエンジンを決定します。
  • USE_X_SENDFILE – 静的ファイルを X-Sendfile を使用して返すかどうか指定します。
  • SECRET_KEY – アプリケーション固有のシークレットキーです。アプリケーションごとに変更してください。
  • SERVER_NAME – ホスト名を指定します。
  • TRAP_HTTP_EXCEPTIONS – hoge
  • TRAP_BAD_REQUEST_ERRORS – huga
  • PRESERVE_CONTEXT_ON_EXCEPTION – aaa
  • SEND_FILE_MAX_AGE_DEFAULT – 静的ファイルのタイムアウトを指定します。
  • PERMANENT_SESSION_LIFETIME – セッションの有効期限を指定します。
  • CACHE_STORE – 標準で使用するキャッシュストアを指定します。
  • CACHE_DEFAULT_TIMEOUT – キャッシュとして保持するデフォルトの時間を指定します。
  • CACHE_THRESHOLD – キャッシュの保持数を指定します。
  • CACHE_KEY_PREFIX – キャッシュとして格納するキーに対して接頭詞を指定します。
  • CACHE_DIR – FileSystemCacheStore を使用する際、キャッシュの保存先を指定します。
  • CACHE_OPTIONS – キャッシュストアインスタンス化時に渡すオプションを指定します。
  • CACHE_ARGS – キャッシュストアインスタンス化時に渡す引数を指定します。
  • LOG_FILE_OUTPUT – ログを外部ファイルとして出力するかどうか指定します。
  • LOG_FILE_ROTATE – ログを外部ファイルとして出力する際、ログローテートするかどうか
  • LOG_ROTATE_MAX_BITE – ログローテートする場合、最大ファイルサイズを指定します。
  • LOG_ROTATE_COUNT – ログローテートする場合、過去の差分として何ファイルまで残すのか指定します。
  • LOG_FILE_DIRECTORY – ログファイルの出力先を指定します。
  • LOG_DEBUG_FORMAT – ログをコンソール上に出力する際のフォーマットを指定します。
  • LOG_OUTPUT_FORMAT – ログをファイルに出力する際のフォーマットを指定します。
